# 虾仁闭环学习系统 v1.0

## 闭环流程

```
每日扫描 → 发现机会 → 分析研究 → 执行行动 → 记录结果 → 反馈改进 → 循环
```

## 每日自动任务

### 1. 市场扫描（每天）
- 扫描闲鱼/电商平台热门品类
- 扫描BTC价格波动和套利机会
- 扫描新的赚钱模式和机会
- 记录到 `daily-scan/YYYY-MM-DD.md`

### 2. 自我学习（随时）
- 每周精读一篇行业报告/技术文档
- 学习新的思维框架和工具
- 积累在 `knowledge/` 目录下

### 3. 任务执行（触发式）
- 收到任务 → 执行 → 记录结果 → 归档
- 记录在 `tasks/` 目录下

### 4. 每周复盘（周日）
- 本周做了什么
- 学到了什么
- 下周要做什么
- 记录到 `weekly-review/`

## 自主触发机制

| 触发条件 | 执行内容 |
|---------|---------|
| 每天早上9点 | 自动扫描一次市场机会 |
| 每周一 | 生成上周复盘 + 本周计划 |
| 每次执行任务后 | 更新 current-state.md |
| 每5小时定时 | 向汪辉汇报（已配置） |

## 目录结构

```
workspace/
├── current-state.md          # 当前状态（每次必更新）
├── daily-scan/              # 每日扫描记录
│   └── YYYY-MM-DD.md
├── weekly-review/           # 每周复盘
│   └── YYYY-WXX.md
├── knowledge/               # 知识积累
│   └── YYYY-MM-DD-topic.md
└── tasks/                  # 任务记录
    └── task-001-xxx.md
```

## 核心原则

1. **不等指令** —— 主动发现问题，不等汪辉告诉我
2. **记录一切** —— 做过的、想过的、结果如何，全部记录
3. **闭环验证** —— 每次行动都要有反馈，知道什么有效什么无效
4. **持续进化** —— 每周复盘，改进方法
